# CineMatch Intelligence Frontend

Production-quality frontend for a MovieLens 20M recommendation system, built with:

- Next.js App Router
- React + TypeScript
- Tailwind CSS
- shadcn/ui-style reusable components
- lucide-react
- TanStack Query (React Query)
- Zod + React Hook Form
- Recharts (for evidence contribution visualization)

This app is wired to the backend API (no mock data) and focuses on clear recommendation storytelling:
user profile, recommendation lists, explanation evidence, and evaluation feedback loop.

## 1. Setup From ZIP (Local Run)

If you received this project as a `.zip`, follow these steps first:

1. Extract the zip and open a terminal in the `front/` folder.
2. Install Node.js 20+ (includes `npm`). Check:

```bash
node -v
npm -v
```

3. Install dependencies:

```bash
npm install
```

4. Create local environment file (this file is not committed):

```bash
cp .env.example .env.local
```

5. Make sure `.env.local` contains:

```env
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
```

6. Start the backend API at the same URL (`http://localhost:8000`) or update `.env.local` to your backend URL.
7. Run the frontend:

```bash
npm run dev
```

8. Open `http://localhost:3000`.

Notes:
- Do not create `node_modules/` or `.next/` manually; they are generated by `npm install` and `npm run dev`.
- Keep `.env.local` local to your machine; do not upload it to git.

## 6. Routes

- `/login` - identity selection and custom-user creation
- `/dashboard` - profile, stats, recommendations, ratings history, model info, evaluations
- `/users/new` - guided custom-user onboarding flow
- `/movies` - movie search/browse
- `/movies/[movieId]` - movie details + neighbors
- `/recommendations/[movieId]` - deep explanation view for a recommendation

## 7. Architecture Decisions

### App Router + Client/Server Boundaries
- App Router is used for route composition and layouting.
- Interactive pages are client components where session state and React Query are required.
- Shared shell and route groups keep a clean navigation structure.

### API Layer + Defensive Parsing
- `lib/api/client.ts` centralizes fetch, base URL, errors, and query params.
- `lib/api/endpoints.ts` maps each backend operation to typed functions.
- `lib/utils/guards.ts` normalizes snake_case/camelCase and varied payload wrappers into stable domain models.

### Domain-First Typed Models
- `types/api.ts` contains request contracts.
- `types/domain.ts` contains UI-oriented entities (users, ratings, movies, recommendations, explanations, evaluations).
- UI components consume domain models, not raw backend payloads.

### Server State + Session State
- React Query handles async data, cache, invalidation, and mutation workflows.
- Zustand (`lib/store/session-store.ts`) stores active user identity with persistence.

### UX / Design System
- Reusable primitives in `components/ui` and feature modules by domain.
- Dark cinematic palette, subtle gradients, and motion for demo-ready visual quality.
- Strong loading, empty, and error states across all key workflows.

## 9. Demo Flow Recommendation

1. Open `/login`
2. Pick MovieLens user or create a custom user
3. If custom user, go through `/users/new` onboarding and add ratings
4. Open `/dashboard` and generate recommendations
5. Open a recommendation explanation (`/recommendations/[movieId]`)
6. Submit evaluation feedback and observe it in evaluation history
